# CLAUDE.md — MindAttic.Ideas.Library (MAIL)

How to work in this repo. Project facts and laws live in the **Codex** docs, not here; this file points at
them and states the working rules.

## Codex — the source of truth

This repo follows the MindAttic Codex documentation standard. Read these before changing anything:

- **[docs/BIBLE.md](docs/BIBLE.md)** — L0, the single source of truth: what the library IS / is NOT, the
  architecture canon, and the Laws (`{#MAIL-LAW-n}`). Stable section ids `{#MAIL-§N}`.
- **[docs/AMENDMENTS.md](docs/AMENDMENTS.md)** — L1, append-only change log. **An amendment wins over the
  bible.** Never rewrite one; supersede it.
- **[docs/USER_STORIES.md](docs/USER_STORIES.md)** — L2, test/proof-cited stories (`MAIL-US-<Epic><n>`).
- **[docs/data/components.json](docs/data/components.json)** — L5, canon-as-data: the catalog of every
  shipped `.idea` (key, kind, version, assembly, artifact, composition). Validated by its
  [schema](docs/data/_schema/component.schema.json). **Cite components by `id`; never restate their fields.**
- **[docs/rfc/](docs/rfc/)** — design notes that graduate into L0 + L2.
- **[../MindAttic.HouseRules.md](../MindAttic.HouseRules.md)** — org-wide laws this repo inherits by
  reference (do not restate or modify).
- **docs/BIBLE.digest.md** — GENERATED by `tools/codex.ps1`; never hand-edit. Injected each session by
  `.claude/hooks/inject-digest.ps1`.

### Rules of engagement
- **Single home per fact.** A fact lives in exactly one layer; everything else links to it by id.
- Reference things by **stable id/anchor**, never by line number.
- When canon changes: edit BIBLE.md (or add an amendment), update the catalog/stories, then run
  `tools/codex.ps1 digest` and `tools/codex.ps1 doctor`.
- Mark a story `✅` only when a build or observed demo proves it (see [MAIL-§8](docs/BIBLE.md#MAIL-§8) and
  [HOUSE-LAW-8](../MindAttic.HouseRules.md)); otherwise `🟡`/`⬜`.

## Tooling

```pwsh
powershell -ExecutionPolicy Bypass -File tools\codex.ps1 doctor   # validate the docset (CI gate)
powershell -ExecutionPolicy Bypass -File tools\codex.ps1 digest   # regenerate docs/BIBLE.digest.md
```

(`pwsh` is not installed on this machine; use Windows PowerShell 5.1 via `powershell`. The script is 5.1-safe.)

## Build & pack

- Build one component: `dotnet build -c Release Widgets/HelloWorld` (or any `Themes/…`, `Controls/…`).
- Build everything: `dotnet build -c Release MindAttic.Ideas.Library.slnx`.
- Pack a `.idea`: see the command in [README.md](README.md). Frozen page source under `Pages/_wip/` is **not**
  built or packed.
- Do not add a `ProjectReference` other than Abstractions; compose components by string id
  ([MAIL-LAW-3](docs/BIBLE.md#MAIL-LAW-3), [MAIL-LAW-4](docs/BIBLE.md#MAIL-LAW-4)).
